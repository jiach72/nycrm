# 变更：移动端与交互体验优化

## 为什么
当前项目已有基础的移动端响应式设计（断点 768px, 1024px），但在触控交互、点击区域、动画流畅度、以及一些 UI 细节方面仍有优化空间。本提案旨在提升移动用户体验，增强交互反馈，并打磨 UI 细节，使整体体验更加专业流畅。

## 变更内容

### 1. 移动端优化
- **触控友好的点击区域**: 确保所有可点击元素的最小尺寸为 44×44px（Apple 人机界面指南）
- **安全区域适配**: 为刘海屏和底部手势条添加 `env(safe-area-inset-*)` 支持
- **移动端菜单关闭**: 点击导航链接后自动关闭移动菜单
- **滚动平滑**: 添加 `scroll-behavior: smooth` 和触摸滚动优化
- **避免 300ms 点击延迟**: 确保 `touch-action` 配置正确

### 2. 用户交互优化
- **按钮交互反馈**: 添加按压效果（`active` 状态的视觉反馈）
- **加载状态**: 为按钮添加加载/禁用状态样式
- **焦点可见性**: 优化键盘导航的 `:focus-visible` 样式
- **链接悬停效果**: 统一和增强链接的悬停动画

### 3. UI 细节优化
- **渐变文字**: 优化 Hero 区域的渐变文字效果
- **卡片阴影一致性**: 统一卡片悬停时的阴影效果
- **动画曲线**: 使用更自然的动画曲线 (`cubic-bezier`)
- **字体加载优化**: 添加 `font-display: swap` 避免 FOIT

## 影响
- **受影响文件**：
  - `src/assets/styles/main.css` - 全局样式
  - `src/components/Navbar.vue` - 导航栏组件
  - `src/components/Footer.vue` - 页脚
  - `src/views/Home.vue` - 首页
  - `src/views/Services.vue` - 服务页面
  - `src/views/Contact.vue` - 联系页面
- **不涉及重大变更 (Breaking Changes)**
